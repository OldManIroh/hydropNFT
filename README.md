# HydroNFT — Контроллер гидропонной системы на ESP32

ESP32-based контроллер для автоматизации гидропонной установки с датчиками pH, TDS, температуры и уровня воды. Интеграция с Home Assistant через MQTT. Поддержка OTA обновлений.

**Версия прошивки:** 0.5.0

## Структура задач FreeRTOS

| Задача                    | Приоритет | Ядро | Описание                        |
|---------------------------|----------|------|--------------------------------|
| `ads1115_task`            | 4        | 1    | Чтение ADC датчиков            |
| `mqtt_sensor_task`        | 4        | 0    | Публикация всех сенсоров в одном JSON |
| `mqtt_ota_check_task`     | 4        | 0    | Проверка флага OTA             |
| `mqtt_ota_progress_task`  | 4        | 0    | Публикация прогресса OTA (динамическая) |
| `espnow_receiver_task`    | 4        | 0    | Приём ESP-NOW сигналов уровня воды |
| `dht_task`                | 3        | 1    | Чтение DHT датчика            |
| `light_schedule_task`     | 3        | 1    | Управление светом по расписанию |
| `pump_valve_schedule_task`| 3        | 1    | Управление насосом и клапаном (автомат) |
| `wifi_reconnect_task`     | 2        | 0    | Переподключение WiFi после сохранения настроек |
| `ota_task`                | 5        | 0    | Загрузка прошивки (динамическая) |

**Распределение по ядрам:**
- **Ядро 0 (PRO_CPU):** MQTT, OTA, ESP-NOW, WiFi reconnect
- **Ядро 1 (APP_CPU):** Сенсоры (ADS1115, DHT), расписания (свет, насос/клапан)

## Структура проекта

```
main/                       # app_main(), все задачи FreeRTOS
components/
  device_control/           # GPIO, данные DHT, режим AUTO/MANUAL
  ads1115/                  # Драйвер ADS1115 (I2C, медианный фильтр)
  mqtt_client/              # MQTT клиент (публикация, подписки, HA discovery)
  ota_client/               # OTA обновление через HTTPS
  sntp_client/              # Синхронизация времени (NTP)
  log_forwarder/            # Пересылка WARNING/ERROR логов в MQTT
  espnow_receiver/          # ESP-NOW приёмник сигналов уровня воды
  settings/                 # Хранение WiFi/MQTT настроек в NVS
  web_server/               # Встроенный HTTP-сервер + дашборд + REST API
managed_components/         # esp-idf-lib пакеты (ads111x, dht, i2cdev)
```

## Автомат состояний насоса/клапана

Единая задача `pump_valve_schedule_task` управляет насосом и клапаном через 4 состояния:

| Состояние | Время | Насос | Клапан | Условие перехода |
|-----------|-------|-------|--------|-----------------|
| **CIRCULATING** | День | ВКЛ | Закрыт | ESP-NOW: верхний уровень → DRAINING |
| **DRAINING** | День | ВЫКЛ | Открыт | Таймер 5 мин → CIRCULATING |
| **NIGHT_STOP** | Ночь | ВКЛ→ВЫКЛ | Закрыт | ESP-NOW: верхний → NIGHT_IDLE |
| **NIGHT_IDLE** | Ночь | ВЫКЛ | Закрыт | Рассвет → CIRCULATING |

**День:** 08:00–23:59, **Ночь:** 00:00–07:59

## MQTT протокол

### Топики публикации (ESP32 → Home Assistant)

| Топик                       | Описание                        | Формат          |
|-----------------------------|--------------------------------|-----------------|
| `hydro/sensor/data`         | Все сенсоры (единый JSON)      | `{"ph":...,"tds":...,"level":...,"water_temp":...,"temp":...,"humidity":...}` |
| `hydro/switch/pump/state`   | Состояние насоса               | `ON` / `OFF`    |
| `hydro/switch/light/state`  | Состояние света                | `ON` / `OFF`    |
| `hydro/switch/valve/state`  | Состояние клапана              | `ON` / `OFF`    |
| `hydro/switch/mode/state`   | Режим работы                   | `auto` / `manual`|
| `hydro/ota/progress`        | Прогресс OTA (%)               | JSON            |
| `hydro/ota/status`          | Статус OTA                     | Строка          |
| `hydro/firmware/version`    | Версия прошивки                | Строка          |
| `hydro/log`                 | Логи WARNING/ERROR             | Строка          |
| `hydro/control/touch`       | ESP-NOW сигнал уровня воды     | `true` / `false`|

**Формат JSON сенсоров:**
- `ph` — напряжение pH датчика (В)
- `tds` — TDS (ppm)
- `level` — напряжение датчика уровня (В)
- `water_temp` — напряжение датчика температуры воды (В)
- `temp` — температура воздуха (DHT, °C)
- `humidity` — влажность воздуха (DHT, %)
- `null` — если данные не валидны

### Топики подписки (ESP32 ← Home Assistant)

| Топик                     | Команда        | Описание                        |
|---------------------------|----------------|--------------------------------|
| `hydro/switch/pump/set`   | `ON`/`OFF`     | Управление насосом (ручное)     |
| `hydro/switch/light/set`  | `ON`/`OFF`     | Управление светом (ручное)      |
| `hydro/switch/valve/set`  | `ON`/`OFF`     | Управление клапаном (ручное)    |
| `hydro/switch/mode/set`   | `auto`/`manual`| Переключение режима             |
| `hydro/ota/update`        | `START`        | Запуск OTA обновления           |

### Пример MQTT Discovery (Home Assistant)

Устройство автоматически отправляет MQTT Discovery конфигурации при первом подключении. Все сенсоры получают данные из единого JSON-топика через `value_template`.

```json
{
  "name": "Уровень pH",
  "state_topic": "hydro/sensor/data",
  "value_template": "{{ value_json.ph }}",
  "unique_id": "esp32_hydro_ph",
  "device_class": "pH",
  "device": {
    "identifiers": ["esp32_hydro_controller"],
    "name": "Hydroponic system",
    "model": "ESP32",
    "manufacturer": "HydroNFT"
  }
}
```

При каждом reconnect отправляются только подписки и состояния — discovery конфигурации не дублируются (`retain=1`).

## Калибровка датчиков

### ADS1115 (I2C ADC)

#### Канал 0: pH метр

Формула преобразования напряжения в pH:
```
pH = (voltage - PH7_VOLTAGE) / (PH4_VOLTAGE - PH7_VOLTAGE) * 3.0 + 7.0
```

Калибровка:
1. Поместите датчик в раствор с pH 7.0 (дистиллированная вода)
2. Измерьте напряжение через топик `hydro/sensor/data` → поле `ph`
3. Поместите датчик в раствор с pH 4.0
4. Измерьте напряжение через `hydro/sensor/data` → поле `ph`
5. Установите константы в `mqtt_client.c`:
```c
#define PH7_VOLTAGE  2.5f   // Напряжение при pH 7
#define PH4_VOLTAGE  3.0f   // Напряжение при pH 4
```

#### Канал 2: TDS метр

Формула:
```
TDS (ppm) = (voltage / TDS_MAX_VOLTAGE) * 1000
```

Калибровка:
1. Измерьте напряжение в растворе с известным TDS (через `hydro/sensor/data` → поле `tds`)
2. Рассчитайте `TDS_MAX_VOLTAGE`:
```
TDS_MAX_VOLTAGE = voltage * 1000 / known_TDS_ppm
```
3. Установите значение в `mqtt_client.c`:
```c
#define TDS_MAX_VOLTAGE_V  2.3f  // Макс напряжение при 1000 ppm
```

#### Канал 3: Уровень воды

Формула (линейная интерполяция):
```
level (%) = (voltage - MIN_VOLTAGE) / (MAX_VOLTAGE - MIN_VOLTAGE) * 100
```

Где:
- `MIN_VOLTAGE` — напряжение при пустом баке (dry sensor)
- `MAX_VOLTAGE` — напряжение при полном баке (sensor в воде)
- Измеряется через `hydro/sensor/data` → поле `level`

### DHT11/DHT22

DHT датчик не требует калибровки. Для повышения точности:
- Используйте DHT22 вместо DHT11
- Обеспечьте циркуляцию воздуха вокруг датчика
- Избегайте прямых солнечных лучей

## OTA обновления

### Процесс обновления

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ MQTT брокер  │────▶│ OTA флаг     │────▶│ OTA задача   │────▶│ Загрузка     │
│ (START)      │     │ установлен   │     │ запущена     │     │ прошивки     │
└──────────────┘     └──────────────┘     └──────────────┘     └──────┬───────┘
                                                                        │
                           ┌──────────────┐     ┌──────────────┐         │
                           │ Устройство   │◀────│ Верификация  │◀────────┘
                           │ перезагруж.  │     │ и запись     │
                           └──────────────┘     └──────────────┘
```

### Запуск OTA

1. Подготовьте файл прошивки на HTTP сервере:
   ```
   http://your-server/firmware/hydropNFT.bin
   ```

2. Отправьте команду START в топик `hydro/ota/update`:
   ```bash
   mosquitto_pub -h 192.168.x.x -t hydro/ota/update -m "START"
   ```

3. Мониторинг прогресса:
   ```bash
   mosquitto_sub -h 192.168.x.x -t hydro/ota/progress
   ```

### Статусы OTA

| Статус              | Описание                                      |
|--------------------|-----------------------------------------------|
| `idle`             | Ожидание команды                              |
| `started`          | Обновление началось                           |
| `downloading`      | Загрузка прошивки                             |
| `completed`        | Обновление успешно завершено                  |
| `failed`           | Ошибка обновления                             |
| `already_in_progress` | Обновление уже выполняется                  |

### Безопасность при OTA

Перед записью новой прошивки:
1. Автоматически отключаются насос, свет и клапан
2. Задача ADS1115 приостанавливается
3. После успешного OTA запускается диагностика
4. Если диагностика не пройдена — откат к предыдущей версии

### OTA через веб-интерфейс

1. Откройте `http://hydronft.local/settings`
2. В разделе «Система и OTA» нажмите «Обновить прошивку»
3. Подтвердите действие — устройство загрузит прошивку с URL из настроек
4. После загрузки устройство перезагрузится автоматически

## ESP-NOW

ESP-NOW используется для приёма сигналов уровня воды от внешнего ESP32.

**Важно:** приёмник ESP-NOW работает на **канале Wi-Fi роутера** (без принудительной смены канала). Это обеспечивает стабильное STA-подключение к роутеру. Передатчик ESP-NOW должен быть настроен на тот же канал, что и роутер.

**Рекомендация:** зафиксируйте канал роутера (1, 6 или 11) вместо «auto», чтобы ESP-NOW работал стабильно.

## Безопасность

| Мера защиты | Описание |
|-------------|----------|
| **AP с паролем WPA2** | Точка доступа `HydroNFT_XXYY` защищена паролем (по умолч.: `hydronft12345678`) |
| **CSRF защита** | Все POST-запросы проверяют Referer + Content-Type — HTML-формы не могут выполнить действия |
| **Парсинг хоста Referer** | Проверяются приватные диапазоны IP (10.x, 172.16-31.x, 192.168.x, 127.x) |
| **JSON экранирование** | Спесимволы в SSID/MQTT URI экранируются перед вставкой в JSON |
| **Case-insensitive state** | API принимает ON/on/On, OFF/off/Off, OPEN/CLOSE, TRUE/FALSE |
| **Валидация Content-Length** | Используется `req->content_len` вместо парсинга заголовка через `atoi()` |

## Устранение неполадок

### ADS1115 не отвечает

1. Проверьте подключение I2C (SDA → SDA, SCL → SCL)
2. Убедитесь, что ADDR пин на ADS1115 подключен к GND (адрес 0x48)
3. Проверьте напряжение (должно быть 3.3В или 5В)

### MQTT не подключается

1. Проверьте IP адрес брокера в menuconfig
2. Убедитесь, что Wi-Fi подключен
3. Проверьте логи: `I (xxxx) MQTT_CLIENT: Connected to mqtt://...`

### OTA зависает

1. Проверьте доступность URL прошивки
2. Убедитесь, что сертификат валиден (если используется HTTPS)
3. Проверьте свободное место в flash

### Веб-интерфейс недоступен

1. Убедитесь, что устройство подключено к Wi-Fi (AP режим: `HydroNFT_XXYY`, пароль: `hydronft12345678`)
2. Попробуйте открыть `http://hydronft.local` или IP адрес устройства
3. Проверьте что веб-сервер включён (menuconfig → Web Server)
